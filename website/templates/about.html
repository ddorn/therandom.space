How does it work ?

All my generators work in the same way. The words or sentences are generated in two steps:

  • I gather a large amount of the things I want to generate and produce a probability table
  • I follow the probability table to create some new words / sentences.

The probability table works as follow: for a given integer D, and a word/letter W, it gives the probability that W is the next word considering the D previous. The probability comes from the sentences/words is the same that is observed in the sentences/words I've gathered in the first step.

For instance, when I generate the proverbs, I set D=2, so the table gives what third word can follow any two words, and with which probability. Therfore, the generation process is simple: I randomly pick two words, and add a third one according to the table, then a fourth word according to the 2nd and 3rd and so on...

For more details, you can find the source code on Gitlab. The idea is very simple, but the code is a bit more complex because I wanted it to work with any depth and be very generic (the exact same code generates words or sentences!). It means that the probability table is a D dimensionnal array (in practice, a dict), generic over D and most functions are recursive.
